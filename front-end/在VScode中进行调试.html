# Debugger for Chrome

就是普通的一个断点工具，但是集成在了 vsCode 里面，比浏览器更加直观，更加方便一点。以前这个插件叫做 Debugger for chrome 或者 Firefox,现在不用了装了，直接编辑器左边导航栏就有。就是那个三角形加甲虫的 logo. ​

# 环境配置

先点开配置文件设置 ​

一般会生成一个默认的 launch.json 文件，直接在上面改就好了,下面例子是可以直接在 vue 脚手架在 chrome 内运行的配置(下面那个)，放在属性上会有提示的

{
    "version": "0.2.0", //默认的0.2.0版本
    "configurations": [
        {
            "type": "chrome",
            "request": "launch",
            "name": "vuejs: chrome",
            "url": "http://localhost:8082",
            "webRoot": "${workspaceFolder}"
        },
        {
            "type": "chrome",
            "request": "launch", //配置文件的请求类型
            "name": "vue-cli", //这个文件的名字
            "url": "http://localhost:8082", //服务运行的位置
            "webRoot": "${workspaceFolder}/src", //目录或者需要执行的文件，值代表vscode打开的目录
            "breakOnLoad": true,
            "sourceMapPathOverrides": {
                "webpack:///src/*": "${webRoot}/*" //VSCode用来定位调试源文件的配置信息。设置了这个属性，VSCode才会在工作区目录下的特定位置寻找源文件，并将其关联到调试中。
            }
        }
    ]
}

​

config 文件配置

configureWebpack: config => {
    config.devtool = 'source-map'
}

配置了这个之后，运行的时候将一个被压缩的文件中的代码对应回其源文件相应的位置。确保你可以在一个应用中调试。 ​

# 在 vue-template(前端模板)项目下兼容

在项目模板下配置了一段代码

config.plugins.push(
    new TerserPlugin({
        test: /\.js(\?.*)?$/i,
        cache: true,
        parallel: 4,
        terserOptions: {
            compress: {
                drop_console: true,
                drop_debugger: false,
                pure_funcs: ['console.log'],
            },
        },
        extractComments: false,
    })
)

需要加个条件，因为映射后文件会被上面的正则文件匹配到，然后（错误信息和打印都被干掉了），改成在生产环境下在加载这个插件。

# 直接加断点

在某行代码后面直接写下 'debugger'关键字 这样也可以直接生成断点，记得用完得删掉 ​

# 总结

多数时候 console.log,和 vue-devtool 工具这两个方法排错已经够了。相对来说，debugger 比较麻烦，但是用多了也还好，但是他定位更精准，排错更准确。 官方网站：https://cn.vuejs.org/v2/cookbook/debugging-in-vscode.html (opens new window)
