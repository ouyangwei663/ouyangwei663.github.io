    个人理解

    首先在 javascript 里面几乎所有东西都是对象，但是它和他又不是一种面向对象编程的语言，因为它的语法没有类 class 的概念（先别说 es6，那是后来提出来的,算是一种语法糖。）

    # 问题区 ＆ Todolist

    # 一、生成对象

    # 1.构造函数生成对象

    所谓"构造函数"，其实就是一个普通函数，但是内部使用了 this 变量。对构造函数使用 new 运算符，就能生成实例，并且 this 变量会绑定在实例对象上（阮一峰说的，精辟）

    function Person(name, age) {
    this.name = name

    this.age = age

    this.action = function() {
        alert('学习')
    }
}
new Person('oy', '24')

    我们把人归为一类，你我都是一个人，但是有各自的年龄和名字。所以想创建一个对象（有年龄，有名字的时候）我们就可以 new 一个 Person('oy','24') 生成了 一个名叫 oy，年龄为 24 有学习行为的人。年龄和名字都是不同的，但是每个人都有学习的行为。那我们可以把不变的挂载到 prototpyte 里面去（后面讲）

    # 2.es6 class 生成类

    在 ES6 中新增加了类的概念，可以使用 class 关键字声明一个类，之后以这个类来实例化对象。类抽象了对象的公共部分，它泛指某一大类(class)对象特指某一个，通过类实例化一个具体的对象 ​

    class Person {
    constructor(name, age) {
        this.name = name
        this.age = age
    }
    action() {
        alert('学习')
    }
}
new Person('oy', '24')

    和上面意义是一样的，只是写法上不一样而已 ​

    # 二、对象继承

    # 1.构造函数的继承

    现在有个对象

    function Animal(){
  this.species="动物"；
  this.special=function(){alert('会动')}
}

    从上面来看，这是一个动物的对象，会动，属于动物的分类。然后 Person(),一个人，也是一个动物，也会动。那我就可以直接继承 Animal，使得人也有动物的属性。 ​

    (1)第一种方法也是最简单的方法，使用 call 或 apply 方法，将父对象的构造函数绑定在子对象上，即在子对象构造函数中加一行： ​

    function Person(name, age) {
    Animal.apply(this, arguments)

    this.name = name

    this.age = age

    this.action = function() {
        alert('学习')
    }
}

    事实上，apply 和 cll 只是把 this 的指向修改到了 animal 身上从而达到继承的效果 ​

    (2）用 prototpyte 来继承

    function Person(name, age) {
    this.name = name
    this.age = age
    this.action = function() {
        alert('学习')
    }
}
Person.prototyte = new Animal()

    就是在原型链上挂载了动物类的这个实例
